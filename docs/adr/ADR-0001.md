# ADR-0001 — Carga Incremental por Watermark de created_at

**Status:** Aceito  
**Data:** 2026-06  
**Closes:** #38

## Contexto

O pipeline precisa ser executado diariamente pelo Airflow sem reprocessar dados já ingeridos. Com volumes crescendo ao longo do tempo, uma carga full a cada run seria ineficiente.

## Decisão

Todas as tabelas de origem possuem a coluna `created_at` (timestamp com default `NOW()`). O Airflow armazena o maior valor de `created_at` já processado como variável (`watermark_created_at`). A cada execução, a DAG lê apenas os registros com `created_at > watermark`.

## Particionamento por camada

| Camada | Partição | Justificativa |
|---|---|---|
| Bronze | `ingestao_date` | Data de entrada no lake — rastreabilidade de quando o dado chegou |
| Silver / Gold (fatos) | `ano_mes` | Data do evento — otimiza queries analíticas mensais |
| Dimensões Silver / Gold | Sem partição | Volume pequeno, sem benefício de particionar |

## Consequências

- **Positivas:** eficiência, custo menor, runs rápidas após a inicial
- **Negativas:** requer que `created_at` exista em todas as tabelas de origem
- **Risco:** se um registro chegar com `created_at` no passado (late data), será perdido na próxima run

## Implementação

```python
# Na DAG — lê watermark salvo
watermark = Variable.get("watermark_created_at", default_var="1970-01-01 00:00:00")

# Query incremental
query = f"(SELECT * FROM reproducoes WHERE created_at > '{watermark}') AS t"
df = spark.read.jdbc(url=JDBC_URL, table=query, properties=props)

# Atualiza watermark no fim da task
Variable.set("watermark_created_at", novo_watermark)
```
