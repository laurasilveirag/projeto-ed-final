# ADR-0003 — Serving Gold → Postgres

**Status:** Aceito  
**Data:** 2026-06  
**Closes:** #40

## Contexto

O Metabase precisa se conectar à camada Gold para gerar os dashboards. O Metabase não lê Delta Lake diretamente — precisa de um banco SQL.

## Decisão

A Gold "oficial" fica em Delta Lake no MinIO. O passo final da DAG (`gold_postgres`) espelha as 7 tabelas Gold para o schema `gold` no PostgreSQL existente. O Metabase conecta exclusivamente no Postgres.

```
Gold Delta (MinIO) ──espelho DAG──▶ PostgreSQL schema gold ──▶ Metabase
```

## Implementação

```python
# Na DAG — após gold_dimensoes e gold_fatos
def gold_postgres():
    conn = psycopg2.connect(...)
    conn.execute("CREATE SCHEMA IF NOT EXISTS gold;")

    for dim in ["dim_tempo", "dim_usuario", "dim_plano", "dim_artista", "dim_musica"]:
        df = spark.read.format("delta").load(f"/tmp/dag_gold/{dim}")
        df.write.jdbc(url=JDBC_URL, table=f"gold.{dim}", mode="overwrite")

    for fato in ["fato_reproducao", "fato_pagamento"]:
        df = spark.read.format("delta").load(f"/tmp/dag_gold/{fato}")
        df.write.jdbc(url=JDBC_URL, table=f"gold.{fato}", mode="overwrite")
```

## Consequências

- **Positivas:** Metabase funciona sem configuração especial; queries SQL padrão
- **Negativas:** dados duplicados (Delta + Postgres); carga extra no Postgres
- **Risco:** se o espelho falhar, Metabase fica desatualizado — mitigado pelo retry do Airflow
