# ADR-0002 — Gold em Star Schema Dimensional

**Status:** Aceito  
**Data:** 2026-06  
**Closes:** #39

## Contexto

A camada Gold precisa ser otimizada para consultas analíticas do Metabase. O modelo normalizado da Silver (12 tabelas relacionais) não é adequado para queries de BI.

## Decisão

Implementar star schema com 2 tabelas fato e 5 dimensões:

### Fatos

| Tabela | Grão | Medidas |
|---|---|---|
| `fato_reproducao` | 1 play válido (≥30s) | `ms_tocados`, `completou`, `contagem` |
| `fato_pagamento` | 1 transação | `valor`, flag `pago` |

### Dimensões

| Tabela | Grão |
|---|---|
| `dim_tempo` | Dia (com `ano_mes` para agregação mensal) |
| `dim_usuario` | Usuário |
| `dim_plano` | Plano de assinatura |
| `dim_artista` | Artista |
| `dim_musica` | Música (desnormalizada com gênero e álbum) |

## Justificativa

- Star schema simplifica as queries no Metabase (menos JOINs)
- `dim_tempo` no grão dia com coluna `ano_mes` permite agregar por mês sem função de data
- `dim_musica` desnormalizada (inclui gênero e álbum) elimina JOINs extras no dashboard

## Consequências

- **Positivas:** queries simples no Metabase, performance analítica melhor
- **Negativas:** redundância de dados em `dim_musica` (gênero e álbum repetidos)
